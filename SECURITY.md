# Security Policy

这个项目本身不处理密钥和网络请求，但它会影响用户的个人知识库管理方式，因此安全边界很重要。

## 请不要提交

- 真实 Obsidian vault 路径。
- 个人笔记、日记、聊天记录、研究数据或未公开文档。
- API key、token、cookie、账号、邮箱、手机号等敏感信息。
- 能够自动删除、覆盖或外传用户笔记的脚本。

## 报告安全问题

如果你发现规则可能导致误删、批量覆盖、隐私泄露或危险自动化，请通过 GitHub issue 或安全私信渠道报告。报告时尽量使用脱敏示例。

## 默认安全原则

- 删除前必须列清单并等待人工确认。
- 批量移动前必须列来源、目标、数量和覆盖风险。
- 不确定时优先归档，不直接删除。
- AI 不应编造来源、改写原始实验数据或隐藏不确定性。
