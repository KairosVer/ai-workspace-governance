# 维护者说明

这份文档面向仓库维护者，说明如何保持项目可用、可传播、可维护。

## 发布前检查

- [ ] README 能在 3 分钟内说明项目用途。
- [ ] Quick Start 能让新用户直接复制使用。
- [ ] 模板中没有真实个人路径和隐私信息。
- [ ] AI 入口文件只做路由，不写详细规则。
- [ ] 规则文件之间没有明显冲突。
- [ ] CHANGELOG 已更新。
- [ ] GitHub topics 包含 `obsidian`、`knowledge-management`、`ai-agents` 等关键词。

## 文档风格

- 优先写可执行规则，不写抽象口号。
- 示例使用虚构路径。
- 遇到危险操作，明确写出人工确认要求。
- 不绑定某个特定 AI 工具。

## 版本策略

- 修正文案：patch。
- 新增规则文件或目录模板：minor。
- 改变顶层目录默认结构：major 或明确迁移说明。
