# 安全规则

本文件记录全库最高优先级的安全规则。任何整理、迁移、清理、脚本操作都必须遵守。

## 删除确认铁律

任何涉及以下关键字或动作的命令，执行前必须列出清单、说明风险、等待用户确认：

- `rm -rf`
- `rm -r`
- `Remove-Item -Recurse`
- `del /s`
- `rd /s`
- 批量 `git clean`
- `git reset --hard`
- 删除超过 3 个文件或目录
- 删除包含附件、日记、项目、输出或原始资料的目录

确认清单应包括：

- 将删除的路径。
- 文件或目录数量。
- 是否有备份。
- 是否可能影响链接或附件。
- 是否存在更安全的归档方案。

## 删除替代策略

优先移动到 `7-Archive`，而不是直接删除。

推荐流程：

1. 列出拟删除内容。
2. 判断是否可归档。
3. 如可归档，先移动到 `7-Archive`。
4. 等用户确认确实无用后，再考虑删除。

## 批量移动规则

批量移动前必须列出：

- 来源路径。
- 目标路径。
- 文件数量。
- 是否覆盖同名文件。
- 是否影响已有链接。
- 是否需要更新索引或项目首页。

移动后检查：

- 文件是否到位。
- 链接是否需要修复。
- 附件引用是否仍可用。
- 是否需要写操作记录。

## 覆盖保护

写入文件前，如目标文件已存在，必须判断：

- 是新建、追加还是覆盖。
- 覆盖是否会丢失用户内容。
- 是否需要先备份或展示 diff。

AI 不应未经确认覆盖用户手写笔记。

## 编码规则

处理中文 Markdown 时，必须显式使用 UTF-8。

PowerShell 示例：

```powershell
Get-Content -LiteralPath "file.md" -Encoding UTF8
Set-Content -LiteralPath "file.md" -Encoding UTF8
```

跨 PowerShell 版本更稳妥的方式：

```powershell
[System.IO.File]::ReadAllText($file, [System.Text.Encoding]::UTF8)
[System.IO.File]::WriteAllText($file, $content, [System.Text.UTF8Encoding]::new($false))
```

禁止在未确认编码的情况下批量改写中文文件。

## 图片和附件安全

外部图片处理流程：

1. 下载到 `1-Attachment/`。
2. 使用清晰文件名。
3. 正文替换为 `![[filename.ext]]`。
4. 确认图片可用。
5. 再考虑删除原外部链接。

不确定是否仍被引用的附件，不直接删除。

## Wiki-link 规则

vault 内正文链接优先使用 Obsidian wiki-link。

推荐：

```text
[[4-Wiki/2-Concepts/example-concept]]
![[example-image.png]]
```

避免在正文中写本机绝对路径，例如：

```text
D:\example\vault\...
/Users/name/vault/...
```

公开模板、社区文档和操作记录中优先使用相对路径。

## 隐私规则

AI 不应把以下内容写入公开文档、规则模板或 issue：

- 真实姓名、邮箱、账号。
- 本机绝对路径。
- 私有项目名。
- 未公开研究数据。
- 私人日记内容。
- API key、token、cookie。

需要举例时，使用虚构名称和相对路径。

## AI 整理边界

AI 可以：

- 建议目录结构。
- 提炼规则。
- 生成模板。
- 整理格式。
- 创建索引。
- 检查标题、链接、位置。

AI 不应未经确认：

- 批量删除。
- 批量覆盖。
- 批量重命名。
- 批量移动。
- 改写原始数据。
- 编造来源。
- 为了美观删掉关键内容。

## 高风险操作前的固定回复

当 AI 准备执行高风险操作前，应先回复：

```text
我将执行一个高风险操作。请确认以下清单后我再继续：

- 操作类型：删除 / 批量移动 / 覆盖 / 批量改写
- 涉及路径：
- 文件数量：
- 主要风险：
- 更安全替代方案：
```

没有用户确认，不继续执行。
