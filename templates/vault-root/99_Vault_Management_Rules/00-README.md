# Vault 管理规则总览

本目录存放 Obsidian vault 的管理规则。这里不是普通知识区，而是管理全库结构、文件流转、安全边界和 AI 操作行为的规则层。

## 适用范围

本规则适用于：

- 个人知识库。
- 学习、研究、写作、项目推进和输出管理。
- 使用 AI 助手进行整理、迁移、检查、总结和规则维护的 vault。

本规则不要求用户采用某一种具体方法论。它可以与 PARA、Zettelkasten、LYT、日记流、项目管理流并存。

## 阅读顺序

整理文件、让 AI 处理笔记、迁移内容或创建目录前，优先阅读：

1. `01-Vault-Structure.md`：判断内容属于哪个目录。
2. `99-Safety-Rules.md`：涉及删除、批量移动、编码、图片、附件时优先阅读。
3. `09-Operation-Log-Rules.md`：需要记录 AI 操作时阅读。
4. `07-Rule-Update-Rules.md`：修改规则前阅读。
5. `08-AI-Review-Rules.md`：检查标题、位置、链接、重复和孤岛页面时阅读。
6. 根据目标目录读取对应规则。

## 规则优先级

1. 用户当前明确指令优先。
2. `99-Safety-Rules.md` 中的安全规则优先于所有效率和整理需求。
3. AI 入口文件只作为路由，详细规则以本目录为准。
4. 目录职责冲突时，按文件实际用途判断，不按文件名机械判断。
5. 不确定时，先建议，不直接移动、删除或覆盖。

## 核心目录职责

```text
3-Inbox     原始材料和临时想法
4-Wiki      长期知识和可复用理解
5-Projects  项目推进和决策现场
6-Outputs   面向交付的正式或准正式产物
7-Archive   冷存储和不活跃内容
```

## 基础工作流

```text
收集 -> 3-Inbox
理解 -> 4-Wiki
推进 -> 5-Projects
交付 -> 6-Outputs
冷存储 -> 7-Archive
```

判断不清的文件，默认留在 `3-Inbox`。需要清理但不确定能否删除时，移动到 `7-Archive`，不要急着删除。

## AI 操作原则

AI 可以：

- 提出目录结构建议。
- 创建规则、模板、索引和检查清单。
- 整理格式和修复明显错别字。
- 建议迁移路径。
- 生成检查报告。

AI 不应：

- 未经确认批量删除文件。
- 未经确认批量重命名或移动大量文件。
- 覆盖原始资料的关键内容。
- 编造来源、链接、实验条件或引用。
- 把个人隐私写入公开规则或模板。

## 操作记录

AI 的整理、迁移、检查和规则更新，不写入单一总日志，而是在以下目录中新建单次操作记录：

```text
99_Vault_Management_Rules/AI_Operations_Log/
```

操作记录使用普通相对路径，不使用 Obsidian 双链，方便在任何编辑器中阅读。
