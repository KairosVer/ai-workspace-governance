# Knowledge Rules

`02_Knowledge` stores reusable understanding.

A file belongs here when it explains, generalizes, or preserves something useful beyond one immediate task.

Good knowledge files include:

- concepts
- methods
- references
- lessons learned
- decision rationale
- reusable checklists

Do not put raw dumps, unfinished transcripts, or one-off task notes here.

Each knowledge file should ideally include:

- the core idea
- source or origin
- scope and limits
- related projects or outputs
