# Workspace Map

This map explains how to move through the workspace.

## Lifecycle Areas

- [01_Inbox](../../01_Inbox/00_Guide/Inbox-Guide.md): raw input and unprocessed material.
- [02_Knowledge](../../02_Knowledge/00_Guide/Knowledge-Guide.md): reusable understanding.
- [03_Projects](../../03_Projects/00_Guide/Projects-Guide.md): active execution.
- [04_Outputs](../../04_Outputs/00_Guide/Outputs-Guide.md): deliverables and publishable artifacts.
- [05_Archive](../../05_Archive/00_Guide/Archive-Guide.md): inactive material worth keeping.

## Governance Areas

- [99_Workspace_Rules](../../99_Workspace_Rules/00-README.md): structure, safety, review, and audit rules.
- [AI operation logs](../../99_Workspace_Rules/AI_Operations_Log/README.md): one file per meaningful AI action.

## Map Rule

Maps organize discovery paths. They should not duplicate full documents or become another copy of the folder tree.
