# Contributing

欢迎提交 issue 和 pull request。这个项目的目标是提供一套清晰、可执行、可迁移的 Obsidian vault 管理规则，而不是绑定某个个人工作流。

## 适合贡献的内容

- 更清楚的规则表述。
- 更稳妥的 AI 操作边界。
- 更通用的目录判断标准。
- 更好的 Obsidian 链接、附件、迁移和归档实践。
- 面向不同使用场景的可选扩展，例如学术、写作、产品、研究、咨询。

## 不适合贡献的内容

- 个人隐私、真实 vault 路径、账号、密钥、聊天记录或未公开资料。
- 只服务单一文件、单一项目、单一职业背景的硬编码规则。
- 鼓励 AI 自动删除、覆盖、批量改写用户笔记的规则。
- 需要特定商业插件才能理解的核心规则。

## 写作规范

- 中文为主，路径和代码块保持 ASCII 优先。
- 一条规则只表达一个动作或边界。
- 规则必须能被 AI 执行或检查。
- 有冲突时修改旧规则，不叠加相互矛盾的新段落。
- 示例使用虚构路径，不使用真实个人路径。

## Pull Request 建议

提交 PR 时请说明：

- 修改了哪些规则文件。
- 为什么需要修改。
- 是否影响目录结构或 AI 入口文件。
- 是否需要用户在迁移前手动确认。
